# Contributing to MyRemoteTouch

First off, thank you for considering contributing to MyRemoteTouch! It's people like you that make MyRemoteTouch such a great tool.

## How Can I Contribute?

### Reporting Bugs
* **Check the FAQ**: Your issue might already be addressed there.
* **Search existing issues**: See if someone else has already reported it.
* **Open a new issue**: Clearly describe the problem, including steps to reproduce and your environment (OS, iOS version, etc.).

### Suggesting Enhancements
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to most MyRemoteTouch users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the [Development Guide](./docs/DEVELOPMENT_GUIDE.md).

## Styleguides

### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less

### Code Style
* **Rust**: Run `cargo fmt` before committing.
* **TypeScript**: Follow the existing indentation and naming conventions (camelCase).
* **CSS**: Use Tailwind classes directly in the components.

## Questions?
Feel free to open an issue with the `question` label.

Thanks for contributing!
